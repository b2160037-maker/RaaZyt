# assets/brand/ — the channel's permanent identity (create ONCE, reuse forever)

These files are the "memory hook" that makes viewers recognize RAAZ FILES.
Drop your own here, **or** run `python make_brand_assets.py` to auto-generate them free.

| File | What it is | Size | How to get it |
|---|---|---|---|
| `signature_sting.wav` | 1.5s audio logo (bass whoosh + heartbeat + whispered name) | — | Auto-generated by `make_brand_assets.py`, or drop your own. |
| `logo_sting.mp4` | ~1.5s animated wordmark clip, prepended/appended to videos | 1920×1080 | Auto-generated, or make one in Canva/CapCut (free) and drop here. |
| `avatar.png` | Channel profile picture | 1024×1024 | Auto-generated (AI), or design your own. |
| `banner.png` | Channel banner art | 2048×1152 | Auto-generated (AI), or design your own. |
| `fonts/` | Bold display fonts for thumbnails | — | Drop `Anton-Regular.ttf` and/or `BebasNeue-Regular.ttf` (both free on Google Fonts). |

## Fonts (recommended, free)
Download and place in `assets/brand/fonts/`:
- **Anton** → https://fonts.google.com/specimen/Anton
- **Bebas Neue** → https://fonts.google.com/specimen/Bebas+Neue

If no font is here, the code falls back to the system DejaVu Bold (installed in the workflow), so thumbnails still work.

## Quick generate everything
```bash
pip install -r requirements.txt
python make_brand_assets.py
```
It skips any file that already exists, so your hand-made assets are never overwritten.

## Fixed branded OUTRO clip (plays at the end of every LONG video only)

Place your outro video here, exactly at:

```
assets/brand/Animate_a_logo_reveal_a_black.mp4
```

(Already added for you. If you ever swap it, keep the same filename, OR set the
`OUTRO_FILENAME` secret/env to the new name.)

Rules the pipeline follows automatically:
- Appended UNMODIFIED as the final ~10 seconds of every LONG video (NOT added to Shorts).
- Never trimmed, cropped, sped up, filtered, or recoloured.
- Long videos: uniformly scaled 1280x720 → 1920x1080 (same 16:9, no crop).
- Its original built-in audio is kept and plays.
- The Subscribe/Like CTA voice (Edge TTS, same channel voice) is mixed ON TOP,
  and the clip's original audio is ducked ~ -6 dB only while the CTA speaks,
  then returns to full. The CTA is auto-fitted to ≤ 10s.
- A QA check confirms the outro is present, full-length, and its audio intact
  before anything uploads.

Commit this file to the repo so GitHub Actions can use it (it is NOT gitignored).

## On-camera HOST clip (used automatically inside LONG videos)

Place your host presenter video here:

```
assets/brand/Animate_this_hooded_host_natu.mp4
```

(Already added for you. To swap it, keep the same filename OR set the `HOST_FILENAME` env/secret.)

How it's used automatically:
- The script marks scenes where the host should be "to camera" (`"host": true`), and the
  intro promise, ending and CTA beats always use the host.
- For those scenes the host clip is shown instead of a stock/AI visual.
- It is **auto-trimmed** (if the scene's narration is shorter than the clip) or
  **auto-looped** (if longer), so it always matches that scene's narration length exactly.
- Scaled 1280x720 → 1920x1080 (16:9, no crop). The clip's own audio is dropped — the
  TTS narration plays over it. Only used in LONG videos (not Shorts).

Commit this file to the repo so GitHub Actions can use it (it is NOT gitignored).
